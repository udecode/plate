# Re-Interview Remaining Scope

## When To Run

- when a restored proof owner exposes a broader or different gap than expected
- when a tranche no longer answers “what next?” cleanly
- when parity and north-star lanes start competing for the next batch

## What This Does

Reframe the remaining work at the level of:

- tranche order
- kept rows
- north-star rows
- owned proof lanes

This is not for small patch reshuffling.

Use it when the shape of the remaining program changed.

The perfect-redesign pivot for `packages/slate` is exactly that kind of
change.
