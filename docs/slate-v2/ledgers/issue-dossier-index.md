# Issue Dossier Index

Read this before opening
[fork-issue-dossier.md](/Users/zbeyens/git/plate-2/docs/slate-v2/ledgers/fork-issue-dossier.md).

The dossier is the long-form fork-local issue record. It is not a first-read
file for normal agent work.

## First-Read Issue Files

1. [issue-coverage-matrix.md](/Users/zbeyens/git/plate-2/docs/slate-v2/ledgers/issue-coverage-matrix.md)
   for fixed, improved, related, and not-claimed rows.
2. [references/pr-description.md](/Users/zbeyens/git/plate-2/docs/slate-v2/references/pr-description.md)
   for maintainer-facing count summaries and exact fixed issue lines.
3. [fork-issue-dossier.md](/Users/zbeyens/git/plate-2/docs/slate-v2/ledgers/fork-issue-dossier.md)
   only for the long-form reasoning behind a specific issue or cluster.

## Dossier Sections

- Clipboard and serialization families.
- Decorations, annotations, and overlay families.
- Browser selection, focus, void, inline, and nested editor families.
- History, undo, operation, normalization, and transform families.
- Mobile, IME, beforeinput, and raw-device proof families.
- Content roots, hidden content, synced blocks, and projection-selection
  families.
- Collaboration and slate-yjs readiness families.

## Agent Rule

Do not load the full dossier to answer "what next?" Start from
`agent-start.md`, `master-roadmap.md`, and this index. Open the dossier only
after naming the exact issue number, cluster, or PR claim that needs long-form
evidence.
