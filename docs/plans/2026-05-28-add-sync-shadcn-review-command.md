- `sync-shadcn` had no command dispatch section, so a future `review` invocation
  could be interpreted as ordinary planning text instead of a staleness audit.
- The right boundary is the source rule because generated skill mirrors are
  Skiller output.

Decisions and tradeoffs:
- `review` is read-only except for a dated review artifact; it must not mutate
  `status.json`, patch `apps/www`, delegate to `task`, or imply user acceptance.
- Staleness is split into upstream, local, status, and ref blockers so the
  command can tell us what is stale instead of returning a vague "rerun sync".
