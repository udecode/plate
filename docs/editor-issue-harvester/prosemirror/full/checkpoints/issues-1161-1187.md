## Workflow Notes

- The first double-space regression was attempted in the `inlines` route, but that route already has visible inline text and punctuation after the read-only inline, making the oracle too weak. The durable test now uses the `mentions` route with a real inline atom followed by plain text.
- The generic `dragTextRange` helper cannot select through rendered mention atom labels because they are not one continuous DOM text node. The `#1164` proof uses real mouse geometry over the final text node after making the mention the leading visible content.
- Safari visual-caret rows were not marked covered by generic inline-void stress because those issues require visual caret placement after exact Safari atom-at-line-end or inserted-atom scenarios.
