---
title: Location
description: API reference for locations in Plate.
---

A Location is either a Path, Point or Range. Methods will often accept a Location instead of requiring only a Path, Point or Range.

```typescript
type Location = Path | Point | Range
```

- [Path](/docs/api/plite/path)
- [PathRef](/docs/api/plite/location-ref#pathref)
- [Point](/docs/api/plite/point)
- [PointRef](/docs/api/plite/location-ref#pointref)
- [Range](/docs/api/plite/range)
- [RangeRef](/docs/api/plite/location-ref#rangeref)

## `LocationApi`

### `isAt`

Check if a value implements the `At` interface.

<API name="isAt">
<APIParameters>
  <APIItem name="value" type="any">
    The value to check.
  </APIItem>
</APIParameters>

<APIReturns type="boolean">
  `true` if the value is either a location or a node.
</APIReturns>
</API>

### `isLocation`

Check if a value implements the `Location` interface.

<API name="isLocation">
<APIParameters>
  <APIItem name="value" type="any">
    The value to check.
  </APIItem>
</APIParameters>

<APIReturns type="boolean">
  `true` if the value is a `Path`, `Point`, or `Range`.
</APIReturns>
</API>

### `isSpan`

Check if a value implements the `Span` interface.

<API name="isSpan">
<APIParameters>
  <APIItem name="value" type="any">
    The value to check.
  </APIItem>
</APIParameters>

<APIReturns type="boolean">
  `true` if the value is a valid `Span`.
</APIReturns>
</API>

## Types

### `Location`

<API name="Location">
<APIAttributes>
  <APIItem name="Path" type="Path">
    An array of numbers representing a node's position.
  </APIItem>
  <APIItem name="Point" type="Point">
    An object with `path` and `offset`.
  </APIItem>
  <APIItem name="Range" type="Range">
    An object with `anchor` and `focus`.
  </APIItem>
</APIAttributes>
</API>

### `Span`

<API name="Span">
<APIAttributes>
  <APIItem name="[0]" type="Path">
    The start path.
  </APIItem>
  <APIItem name="[1]" type="Path">
    The end path.
  </APIItem>
</APIAttributes>
</API>
