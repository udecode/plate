Use these links as starting points. The Slate v2 docs and package READMEs are
the source of truth for current APIs.

## First-Party References

- [Why This Fork](../why-this-fork) explains the public package set,
  browser proof posture, and beta scope.
- [Migration guide](../migration) gives an ordered path for moving an
  existing editor to Slate v2.
- [Docs proof map](docs-proof-map) links public claims to tests and proof
  commands.
- [Slate Browser](../libraries/slate-browser) documents Slate's first-party
  browser proof harness.

## Ecosystem Starting Points

Check Slate v2 support before installing an ecosystem package. Some packages are
still useful as design references even when their released builds target older
Slate APIs.

- [Plate](https://github.com/udecode/plate) is the product-level rich text
  framework built on Slate.
- [`slate-yjs`](https://github.com/BitPhinix/@platejs/yjs/) provides Yjs-oriented
  collaboration utilities for Slate.
- [`slate-angular`](https://github.com/worktile/slate-angular) and
  [`slate-vue3`](https://github.com/Guan-Erjia/slate-vue3) explore non-React
  view layers around Slate.
