## `PlatePlugin`

<APIList
  type="object"
  description={
    <>
      Plate plugins are objects passed to <code>Plate</code>{' '}
      <a href="/docs/plate#plugins">plugins</a> prop.
    </>
  }
>
<APIItem name="key" type="string">

Property used by Plate to store the plugins by key in `editor.pluginsByKey`.

- Unique and required.
- Naming convention:
- all Plate element keys start with `ELEMENT_`.
- all Plate mark keys start with `MARK_`.

</APIItem>
  
<APIItem name="component" type="RenderFunction">

Property used by Plate to render a slate element or leaf.

- React component with element of leaf props.

</APIItem>
  
<APIItem name="decorate" type="Decorate" >

Property used by Plate to decorate editor ranges.

- If the function returns undefined then no ranges are modified.
- If the function returns an array the returned ranges are merged with the ranges called by other plugins.
- See https://docs.slatejs.org/concepts/09-rendering#decorations

</APIItem>

## `createPlugins`

<APIList
  type="function"
  description={
    <>
      Creates a new array of plugins by overriding the plugins in the original array.
      Components can be overridden by key using <code>components</code> in the second param.
      Any other properties can be overridden by key using <code>overrideByKey</code> in the second param.
    </>
  }
  returns={
    <>
      A list of <code>PlatePlugin</code> objects.
    </>
  }
>
<APIItem name="key" type="string">

Property used by Plate to store the plugins by key in `editor.pluginsByKey`.

- Unique and required.
- Naming convention:
- all Plate element keys start with `ELEMENT_`.
- all Plate mark keys start with `MARK_`.

</APIItem>
