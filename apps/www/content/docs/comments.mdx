## Features

The Comments plugin allows users to add comments to text as marks.

These comments can include explanations of what the author or editor did or suggests,
such as questions about unclear statements,
notes on facts or continuity errors, common mistakes, or ideas for expanding a paragraph.

To add a comment, select the text and:

- Click on the comment button to add a comment
- Alternatively, use the hotkey **`⌘+⇧+M`**

## Installation

```bash
npm install @udecode/plate-comments
```

## Usage

```tsx
import {
  createCommentsPlugin,
  MARK_COMMENT,
  PlateCommentLeaf,
} from '@udecode/plate';

const plugins = createPlugins(
  [
    // ...otherPlugins,
    createCommentsPlugin(),
  ],
  {
    components: {
      ...createPlateUI(),
      [MARK_COMMENT]: PlateCommentLeaf,
    },
  }
);
```

```tsx
import { TText, Value } from '@udecode/plate-common';

export interface CommentUser {
  id: string;
  name: string;
  avatarUrl?: string;
}

export interface TComment {
  id: string;

  /**
   * Slate value of the document.
   */
  value: Value;

  /**
   * @default Date.now()
   */
  createdAt: number;

  /**
   * Author id.
   */
  userId: string;

  /**
   * Parent comment id it replies to.
   */
  parentId?: string;

  /**
   * Whether the comment is resolved.
   */
  isResolved?: boolean;
}

export interface TCommentText extends TText {
  comment?: boolean;
  comments?: Record<string, boolean>;
}

export interface CommentsPlugin {
  hotkey?: string | string[];
}

export const MARK_COMMENT = 'comment';
```

### `CommentUser`

An interface representing a user who can make comments.

<APIParameters>
  <APIItem name="id" type="string">
    The unique identifier of the user.
  </APIItem>
  <APIItem name="name" type="string">
    The name of the user.
  </APIItem>
  <APIItem name="avatarUrl" type="string" optional>
    The URL of the user's avatar.
  </APIItem>
</APIParameters>

### `TComment`

An interface representing a comment in the editor.

<APIParameters>
  <APIItem name="id" type="string">
    The unique identifier of the comment.
  </APIItem>
  <APIItem name="value" type="Value">
    The Slate value of the document.
  </APIItem>
  <APIItem name="createdAt" type="number">
    The time the comment was created. Default is the current time
    (`Date.now()`).
  </APIItem>
  <APIItem name="userId" type="string">
    The identifier of the user who made the comment.
  </APIItem>
  <APIItem name="parentId" type="string" optional>
    The identifier of the parent comment this comment replies to.
  </APIItem>
  <APIItem name="isResolved" type="boolean" optional>
    Indicates whether the comment is resolved.
  </APIItem>
</APIParameters>

### `TCommentText`

An interface extending TText with additional properties for managing
comments.

<APIParameters>
  <APIItem name="comment" type="boolean" optional>
    Indicates whether the text includes a comment.
  </APIItem>
  <APIItem name="comments" type="Record<string, boolean>" optional>
    An object containing boolean values for each comment associated with the
    text, with the comment id as the key.
  </APIItem>
</APIParameters>

### `CommentsPlugin`

An interface representing a plugin for handling comments in the editor.

<APIParameters>
  <APIItem name="hotkey" type="string | string[]" optional>
    The hotkey or array of hotkeys to listen for triggering a plugin action.
  </APIItem>
</APIParameters>
