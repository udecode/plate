## Features

- **`createBlockquotePlugin()`** for the `blockquote` element
- **`createCodeBlockPlugin()`** for the `code_block` element
- **`createHeadingPlugin()`** for the `h1`, `h2`,... elements
- **`createParagraphPlugin()`** for the `p` element

These plugins provide the foundational formatting options for structuring your content effectively,
allowing you to create blockquotes, code blocks, headings, and paragraphs with ease.

Use the toolbar buttons to apply blockquote, code block, heading, and paragraph formatting to your text.

## Installation

```bash
npm install @udecode/plate-basic-elements
# or
npm install @udecode/plate-block-quote
npm install @udecode/plate-code-block
npm install @udecode/plate-heading
npm install @udecode/plate-paragraph
```

## Usage

```tsx
import { createBasicElementsPlugin } from '@udecode/plate';

const plugins = createPlugins(
  [
    // ...otherPlugins,
    createBasicElementsPlugin(),
  ],
  {
    components: createPlateUI(),
  }
);
```

The code block component is not included in Plate UI. You can add it using:

```tsx
createPlateUI({
  [ELEMENT_CODE_BLOCK]: CodeBlockElement,
});
```

### Options

```tsx
export type HotkeyPlugin = {
  /**
   * Hotkeys to listen to trigger a plugin action.
   */
  hotkey?: string | string[];
};

export interface CodeBlockPlugin extends HotkeyPlugin {
  syntax?: boolean;
  syntaxPopularFirst?: boolean;
  deserializers?: string[];
}

export interface HeadingPlugin extends HotkeyPlugin {}

export interface HeadingsPlugin {
  /**
   * Heading levels supported from 1 to `levels`
   */
  levels?: number;
}
```

### `HotkeyPlugin`

A plugin used to listen to hotkeys and trigger associated actions.

<APIParameters>
  <APIItem name="hotkey" type="string | string[]" optional>
    A single hotkey or an array of hotkeys to listen for.
  </APIItem>
</APIParameters>

### `CodeBlockPlugin`

A plugin used for handling code blocks.

<APIParameters>
  <APIItem name="hotkey" type="string | string[]" optional>
    A single hotkey or an array of hotkeys to listen for.
  </APIItem>
  <APIItem name="syntax" type="boolean" optional>
    A flag indicating whether syntax highlighting should be applied. If set to
    `true`, syntax highlighting is enabled.
  </APIItem>
  <APIItem name="syntaxPopularFirst" type="boolean" optional>
    If `true`, popular syntax options will be shown first in the syntax
    selection dropdown.
  </APIItem>
  <APIItem name="deserializers" type="string[]" optional>
    A list of deserializers to be used with the plugin.
  </APIItem>
</APIParameters>

### `HeadingPlugin`

A plugin used for handling headings.

<APIParameters>
  <APIItem name="hotkey" type="string | string[]" optional>
    A single hotkey or an array of hotkeys to listen for.
  </APIItem>
</APIParameters>
HeadingsPlugin A plugin used for handling multiple levels of headings.

<APIParameters>
  <APIItem name="levels" type="number" optional>
    Defines the heading levels supported, ranging from 1 to `levels`.
  </APIItem>
</APIParameters>
